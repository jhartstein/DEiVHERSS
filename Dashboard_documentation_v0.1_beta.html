Shiny Dashboard

In a Shiny R Dashboard code, there are typically three fundamental elements.

Step 1: create and fill dashboard.

User interface (UI): this part of the code defines the layout and appearance of the dashboard. It specifies how the dashboard will look to the user, including the arrangement of widgets such as buttons or sliders, and the overall structure of the interface.

The following code defines the interface structure: the dashboard page has the header, the sidebar with the option to choose between data or diagrams by research fields, and the body.

# create and fill dashboard
ui <- dashboardPage(
  dashboardHeader(title = "Field assessments"),
  dashboardSidebar(
    sidebarMenu(
      menuItem("Data", tabName = "datatable", icon = icon("table", lib = "font-awesome")),
      menuItem("All Fields", tabName = "all", icon = icon("graduation-cap", lib = "font-awesome")),
      menuItem("Humanities", tabName = "field1", icon = icon("masks-theater", lib = "font-awesome")),
      menuItem("Social Sciences", tabName = "field2", icon = icon("person", lib = "font-awesome")),
      menuItem("Life sciences", tabName = "field3", icon = icon("syringe", lib = "font-awesome")),
      menuItem("Natural Sciences", tabName = "field4", icon = icon("tree", lib = "font-awesome")),
      menuItem("Engineering", tabName = "field5", icon = icon("car-side", lib = "font-awesome"))
    )
  ),
  dashboardBody(
    tabItems(
      # first tabItem
      tabItem(tabName = "datatable", 
              tabBox(id="t1", width = 20,
                     tabPanel("Data", dataTableOutput("data_output"), icon = icon("table")) 
              )
              
      )

Step 2: create server function.

Server Logic: The server logic is where you define the behaviour and functionality of your dashboard. This part of the R code contains functions and operations that react to user inputs from the UI. It processes data, performs calculations and updates the UI dynamically based on user actions.

This particular piece of the code is responsible for output of the data table and creation of diagrams based on the data for all research fields.

Running the Shiny app: use attached full version of R-code (paths are redacted and must be filled by you) to run the Dashboard and to use it dynamically.
